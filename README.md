## アプリ概要
本アプリは、ネットワークを構成するPC、スイッチ、ルータなどのコンピュータネットワーク機器になりきって、ルーティングテーブル作成、パケットの解析と作成などをGUIを通して体験しネットワークの理解を深める、コンピュータネットワークゲーミフィケーション教育アプリ **「Packet-Crafter(仮称)」** である。

### 目的
コンピュータネットワークゲーミフィケーション教育アプリというジャンルであることから、教育機関及び企業等の研修での利用を第一に想定している。

近年では量子ネットワークや量子暗号、ブロックチェーン、時代のニーズに応じたプロトコルなど新たなネットワーク技術の台頭があり、ネットワークやネットワークセキュリティの実運用が複雑化しているため初学者は教科書と現実のギャップを乗り越えることが困難になっている。

そこで本アプリは、ネットワークの学習を始める者、ネットワークの理解をより深めたい者に対して理解の手助けを第一の目標とし、ネットワークエンジニア、サーバーサイドエンジニア、ネットワークインフラエンジニアなどのネットワークに関わる人材の量・質の強化を望み、**未来のコンピュータネットワークの発展を最終的な目的** としている。

### 想定ターゲット
本アプリのターゲット層は以下のような者である。
- 実際のネットワークの流れを知りたい人
- ネットワークの本を読んだが、実際にどのような通信・処理が行われているのか知りたい人
- 体験を通してネットワークの理解を深めたい人
- 自身が受け持つ講義の学生・生徒や会社の研修に参加する社員にネットワークを理解してもらうための便利なツールがほしい人

### 新規性・差別化
ここでの説明は、決して他の既製品を批判するこを目的としているわけではないため具体的なアプリ名は伏せる。

これまでの既存のアプリケーションは、
- ネットワークを構成してシミュレーション・パケットトレーサーを行う
- パケットを観察し、パケットを作成する
- ネットワーク・インフラエンジニアになりきり、ネットワークの設計・運用・保守を行う

といった機能を持つものが多かった。

このようなアプリケーションもネットワークへの理解を促進させるアプリとして間違いないものであると考えているが、我々はこれらから新たなインスピレーションを感じ、以下の新たな特徴を持つ本アプリの開発を構想した。
- ネットワーク全体を俯瞰しつつパケットの流れを見るのではなく、ネットワーク機器になりきってパケットの受取・作成、転送などを学ぶ
- 通信1つ1つを自らの手で行うことにより、どのネットワーク機器がどの役割を担っているのかを理解できるような仕組みを提供する

このような特徴により、他のアプリとは差別化された新たな価値を創造できると考えている。

### 現在の構想
ユーザはルームを作成する。他のユーザと一緒にプレイする場合は他のユーザはルームID及びパスワードを用いてルームに入室する。

各ユーザはステージごとに登場するネットワーク機器になりきる。例えばルータなら、
- ARPテーブルの作成
- ルーティングテーブルの作成(ステージごとに静的ルーティング・動的ルーティングかどうか設定しておく)
- パケットに含まれるMAC・IPアドレスを参照し、ネクストホップを決定
- 送信するためのパケットを作成するためにARPテーブルやルーティングテーブルを参照し、パケットの決められた箇所にアドレス等の情報を入力する。

本アプリは、開発当初は1つのwebサーバで稼働させるが、最終的にはlinux等のOSの上でVMもしくはコンテナを作成することで各々が自由に運用できるようなセルフホストアプリとして開発を進める予定である。

### 追々の構想
ハッカソン終了後、ないしは現在の構想を開発後に実装したい機能

#### セルフホストアプリ化
前述した通り、各々が自前のサーバにVMもしくはコンテナを建てることによって簡単に運用できるセルフホストアプリ化を目指す。

これにより、小学校から大学までの教育者は学校に設置されておりかつVMなどを構築できるようなサーバにVMもしくはコンテナを構築するだけで生徒・学生に対してネットワークの演習教育を実施することが可能となる。

#### L4以上のプロトコルの実装
後述するサーバの実装も含め、L4以上のプロトコル(TCP/UDPやTLS、IPsec等のVPN、HTTPやSMTP等の多くのアプリケーションプロトコル)を体験できるようなネットワーク機器の追加及び機能の追加を目指す。

より生活に近いプロトコルの通信の流れを理解することによって、ネットワークへの更なる理解が期待できる。

#### サーバ
ネットワーク構成にサーバを追加し、サーバになりきれるようにする。

webサーバであれば、
- TCPハンドシェイクやTLSハンドシェイクを行う
- 届いたパケットを見てポートが80番ないしは443番であるか(適切なポートに届いているか)を見る
- cookie等を用いてセッション管理を行う
- httpリクエストに応じてhttpレスポンスを作成する
- 3層クライアントサーバシステムのwebサーバの場合、DBサーバとの通信も行う

などなど、サーバまで拡張することでより多くの体験をすることができる。

#### ファイアウォール
パケットフィルタリングルールに従い、パケットの通過可否を判断する。また、ファイアウォールの種類に応じてどのような挙動をするのかを観察できるようにするため、サーキットゲートウェイ方式やステートフルパケットインスペクション方式など代表的なファイアウォールの仕組みを実装する。

#### ステージ自作機能
開発初期段階では開発側でステージを用意し、ユーザにはそのステージをクリアしてもらうような方式を採る。
しかしそれでは教育者ごとに「どのようなネットワーク構成を通してどのようなことを学んでほしいか」というニーズに応えることはできない。

したがって、ユーザごとにネットワークを構成し、そのネットワーク上でプレイできるような機能が必要である。

#### パケットなりきり
PCがパケットを作成し送信する。そうするとそのパケットはスイッチ、ルータなどを通り目的次第でサーバや他のPCとの通信を行う。

その過程を体験するために、パケットになりきりパケットの一生を体験するような機能を実装する。

#### 自作プロトコル作成・シミュレーション
ステージ自作機能、基本的なネットワーク機器の実装が終了した後、研究者や知的好奇心が旺盛な者に向けて自作プロトコルの作成及びそのシミュレーション機能を提供する。

これにより、本アプリは単なる教育アプリではなく研究までもカバーするアプリと拡張される。
